NEURALYZER - a differential recommender system that can forget user data very fast.

User interactions

The interaction matrix \( \mathbf{H} \) contains the behavioral data of users. Every row \( \mathbf{H}_i \) represents a single user \( i \) and entries denote which items this particular user likes.

Item cooccurrences

The item cooccurrence matrix \( \mathbf{C} = \; \mathbf{H}_i^\top \mathbf{H}_i \) denotes how many users interacted with a pair of items (e.g., watched both movies).

Item similarities

The item similarity matrix \( \mathbf{S} = \; [C_{ij} / (n_i + n_j - C_{ij})]_{ij} \) is the actual model of our recommender system. Each entry denotes the Jaccard similarity between the user interactions with a pair of items. These similarities are the basis for answering recommendation queries.

Changes computed by Differential Dataflow

Neuralyzer is implemented in Differential Dataflow (DD). This webapp communicates with DD via web sockets. It adds and removes user interactions to and from the recommenders dataflow computation. DD will compute the changes to the intermediate data structures, the model and the ouputs for the recommendation queries, and sent them back to the web app.
